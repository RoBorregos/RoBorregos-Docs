# Path-planing

## Abaout Path-planing
Path planning is one of the most important components in the VSSS strategy system, since it is responsible for generating safe and efficient trajectories for the robots during a match. The implemented system is based on artificial potential fields, where different virtual forces are combined to guide the robot through the environment while avoiding collisions with dynamic and static obstacles.

The algorithm integrates attractive forces toward a desired target, repulsive forces generated by obstacles, vortex-based avoidance behaviors, and wall avoidance. By combining these behaviors, the robots are capable of navigating in highly dynamic situations while maintaining smooth and stable motion. The generated vectors are later transformed into velocity and orientation commands that can be executed by the robot controllers.

##  Objetive
The main objective of the path planning system is to move the robot from its current position to a desired target position while avoiding collisions with enemies, teammates, the ball, and the field boundaries. Since the environment in VSSS changes continuously during gameplay, the algorithm must react in real time to moving obstacles and blocked trajectories.

To achieve this, the system evaluates the environment every cycle and computes a resultant motion vector. When the path toward the target is clear, the robot follows a direct attractive trajectory. However, when an obstacle blocks the desired path, the system switches to a vortex-like navigation behavior that allows the robot to move around the obstacle instead of getting trapped or oscillating in place.

Although different obstacle configurations may appear during a match, the core navigation logic remains the same. The attractive, repulsive, and wall avoidance fields are always active, while the vortex navigation is activated only when an obstacle directly interferes with the path toward the target.

## Atractive field
The attractive field is responsible for guiding the robot toward a target position. This behavior is implemented in the function:

```cpp
attractive_vector(robot, target_x, target_y)
```

The algorithm calculates the displacement between the robot and the target using the difference in coordinates. Then, the vector is normalized so that only the direction is preserved. Finally, the normalized vector is scaled using constants.ATTRACTIVE_GAIN, generating a force that pulls the robot toward the objective.

This attractive component represents the primary navigation behavior of the robot and ensures that the robot continuously attempts to reach its assigned target. 

## Obstacle blocking detection
The function:

```cpp
is_obstacle_blocking(entity, obstacle, target_x, target_y)
```

Determines whether an obstacle is directly interfering with the path toward the target. The algorithm projects the obstacle position onto the direction vector between the robot and the target. If the obstacle lies within the trajectory corridor and inside a safety margin, it is considered blocking.

## Repulsive field
Obstacle avoidance is implemented through the function:

```cpp
repulsion(robot, obstacle, influence_radius, canvas=None)
```

The repulsive field generates a force that pushes the robot away from nearby obstacles. The repulsion strength depends on the distance between the robot and the obstacle and decreases exponentially as the distance increases.

The closer the robot gets to an obstacle, the stronger the repulsive force becomes. Outside the influence radius, the obstacle has no effect on the trajectory generation. This behavior allows smooth obstacle avoidance while preventing abrupt changes in movement.

## Vortex navigation
When an obstacle directly blocks the trajectory toward the target, the planner activates a vortex-based navigation behavior using the function:

```cpp
rolling_vector(robot, obstacle, target_x, target_y)
```
Instead of simply pushing the robot away from the obstacle, the algorithm generates a tangential force that causes the robot to move around the obstacle. The direction of the tangential motion is determined by the function:

```cpp
approach_side
```
Which uses the sign of a cross product to determine whether the obstacle should be bypassed from the left or right side. The final motion is generated by combining the tangential vortex force with a radial repulsive force. This approach prevents the robot from getting stuck in local minima, a common problem in traditional potential field methods.

## Wall avoidance

The function:

```cpp
wall_repulsion(robot)
```
prevents robots from colliding with the boundaries of the field. The algorithm calculates the distance between the robot and each wall and generates repulsive forces whenever the robot enters a predefined safety margin near the field limits. This behavior ensures that robots remain inside the playable area and maintain stable trajectories even near the edges of the field.

## Resultant field generation

The main planner is implemented in the function: 

```cpp
field(robot, target_x, target_y, enemies, teammates, ball, attacking_right=True)
```
This function combines all the generated forces into a single resultant vector. For every enemy, teammate, and the ball, the planner first checks whether the object blocks the trajectory. If blocking is detected, the vortex navigation behavior is used; otherwise, standard repulsion is applied. After processing all obstacles, the attractive field and wall repulsion are added to the total vector.

Finally, the resultant vector magnitude is limited using 
```cpp
constants.MAX_FIELD 
```
to avoid excessive speeds and unstable movements. The resulting vector represents the final desired motion of the robot.

## Motion vector conversion

The function 
```cpp
resultant_vector(fx, fy, base_speed)
```
converts the resultant field vector into movement commands. The magnitude of the vector is interpreted as the desired speed, while the direction is computed using 
```cpp
math.atan2(fy, fx)
```
The final speed is clamped to the robot’s maximum allowed velocity, ensuring safe and controllable motion. This function acts as the interface between the path planning system and the robot motion controller.
